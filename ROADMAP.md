# Product Roadmap: Toward Pilot/Beta

The goal is to move from a basic prototype to a "usable state" where the developer can run a pilot program and eventually sell the platform.

## Phase 1: Pilot Ready (Current Focus)
- [x] **Responsive UI**: Implemented mobile-first table-to-card transitions for all major views.
- [ ] **Refine Order Flow**: Simplify the process of adding/editing items.
- [ ] **Order Management**: Improve how bakers see and manage current orders.

## Phase 2: Pilot Enhancements
- [ ] **Order Status tracking**: (Deferred from Phase 1) Track states like Paid, Picked Up, etc.
- [ ] **Complete Order step**: Add a formal confirmation step if needed.
- [ ] **Store Management Improvements**:
    - [ ] Dashboard for bakers to see all orders for an event at once (for prep).
    - [ ] Export orders (CSV) for printing/prep.

## Phase 2: Beta Features
- [ ] **Payments**: Integration with Stripe for upfront payment (optional but recommended for subscription model).
- [ ] **Notifications**: Improve email notifications for order confirmation and reminders.
- [ ] **Multiple Events**: Better handling of recurring events.

## Phase 3: Commercialization
- [ ] **Subscription Model**: Logic for charging bakery owners.
- [ ] **Custom Domains**: Allow bakers to point their own domains.
- [ ] **Advanced Analytics**: Prep lists, ingredient calculators, etc.
